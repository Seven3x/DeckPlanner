你是一个资深 Python 游戏 AI/架构工程师。请帮我设计并实现一个“单机本地研究用途”的《杀戮尖塔2》自动出牌原型，不考虑联网、对战、反作弊绕过，也不要涉及任何破坏游戏或注入游戏进程的内容。目标是：先做一个可扩展的决策引擎与状态模拟框架，重点解决“复杂卡牌特效如何表示与估值”的问题，而不是一开始追求完美胜率。

## 总目标
请用 Python 写一个结构清晰、易扩展、便于后续继续补牌和补机制的原型工程。核心思想不是“给每张牌写一个固定分数”，而是：

1. 把卡牌效果表示成可执行的规则/状态转移
2. 在当前状态下评估动作，而不是静态评估卡牌
3. 支持跨回合效果、触发器、重放/复制、与其他卡牌联动
4. 先做“规则引擎 + 启发式评估 + 有限深度搜索”的方案
5. 不依赖神经网络；先把架构搭好，后续再考虑 RL/监督学习

## 你需要产出的内容
请一次性给我这些内容：

### A. 架构设计

### B. 关键设计原则
必须遵循这些原则：

1. **状态优先，不做静态卡牌打分**
   - 评价的是 Q(s, a) 或“当前状态下动作 a 的价值”
   - 不要把一张卡永久写成固定 8 分、10 分这种

2. **效果用中间表示（IR）或原子操作表示**
   - 一张卡由一组原子操作组成
   - 例如：
     - DealDamage
     - GainBlock
     - DrawCards
     - GainEnergy
     - ApplyBuff
     - ApplyDebuff
     - AddTrigger
     - ScheduleEffect
     - ReplayCard / CopyNextCardEffect
     - Conditional
   - 复杂牌通过组合这些操作实现，而不是写一堆分散 if/else

3. **事件驱动**
   - 支持以下事件：
     - on_turn_start
     - on_turn_end
     - on_card_played
     - on_attack_played
     - on_skill_played
     - on_damage_taken
     - on_block_gained
     - on_draw
     - on_discard
     - on_exhaust
   - 触发器要能设置持续时间、触发次数和失效条件

4. **跨回合效果必须显式表示**
   - 例如“下回合获得 2 能量”不能直接折算为当前分数
   - 必须进入 pending_effects / delayed queue

5. **联动靠状态与标签表达**
   - 例如：
     - 本回合已打出几张攻击牌/技能牌
     - 是否存在某个 buff
     - 弃牌体系是否成型
     - 下一张攻击翻倍
   - 不要为每种联动都单独写特殊估值函数

6. **可搜索**
   - planner 要能对当前手牌进行有限深度搜索
   - 优先实现 DFS / Beam Search
   - 可选支持 rollout
   - 搜索对象是“本回合出牌序列”

### C. 先实现一个最小可运行版本（MVP）
请先实现一个能运行的 MVP，不追求完整游戏覆盖，但要把关键机制打通。MVP 至少支持：

- 玩家基础状态：hp, energy, block, buffs, debuffs
- 手牌 / 抽牌堆 / 弃牌堆 / 消耗堆
- 单个敌人，基础属性：hp, block, intent_damage
- 出牌
- 抽牌
- 回合开始 / 结束
- 若干基础原子效果
- 单次触发器
- 延迟到下回合的效果
- 一个简易评估函数
- 一个简易搜索器

### F. 评估函数设计
请实现一个启发式评估函数 evaluate_state(state)，不要太复杂，但要体现分层思想。至少考虑：

1. 生存价值
   - 当前血量
   - 当前格挡
   - 是否能扛住敌人下回合意图伤害

2. 输出价值
   - 敌方剩余血量下降
   - 是否接近击杀

3. 资源价值
   - 剩余能量是否浪费
   - 手牌质量
   - 抽牌潜力

4. 引擎价值
   - 是否建立了未来触发器
   - 是否挂上了下回合收益
   - 是否形成“下一张攻击增益”之类的状态

请把权重写得比较清楚，便于后续调参。

### G. 搜索器
请实现一个简单但实用的规划器：

- 输入：当前 GameState
- 输出：推荐的本回合出牌序列，以及最终评分
- 做法：
  - 枚举当前合法动作
  - 模拟出牌后的新状态
  - 递归搜索到一定深度，或直到不能出牌
  - 用启发式评估叶子状态
- 为了控制爆炸：
  - 支持 max_depth
  - 支持 beam_width（可选）
  - 对重复状态做简单剪枝（可选）
- 输出时打印：
  - 最佳序列
  - 每一步模拟后的状态摘要
  - 最终分数
